# VM Control Web App

A modern web application for controlling and monitoring virtual machines in Azure. Built with Nuxt 4, Vue 3, and deployed as an Azure Static Web App with serverless Azure Functions backend.

## Setup

### Installation

1. **Clone and install**

   ```bash
   git clone <repo-url>
   cd tboersma-dev-site
   npm install
   cd api && npm install && cd ..
   ```

### Local Development

The dev command has been set up to build the api, start the dev server, and start the local Azure Functions

```bash
npm run dev
```

### Building for Production

The build command is also configured to build the local Azure Functions as well

```bash
# Build frontend and API
npm run build
```

## Description

### 🎯 Features

- **Real-time VM Status Monitoring** – Check if your VM is running or stopped with automatic polling
- **One-Click VM Control** – Easily turn your VM on or off with visual feedback
- **Smart Polling** – Adaptive polling (normal vs. fast) for responsive UX
- **Error Tracking** – Automatic error logging to Azure Application Insights
- **Time-based Overrides** – VM start requests have a 4-hour automatic override window
- **Responsive Design** – Works on desktop and mobile devices
- **Type-Safe Backend** – Built with TypeScript for reliability

## 📋 Tech Stack

### Frontend

- **Nuxt 4** – Full-stack Vue framework
- **Vue 3** – Reactive UI components
- **PrimeVue** – Component library
- **TailwindCSS** – Utility-first CSS
- **Axios** – HTTP client
- **Pinia Colada** – Hooks for queries
- **TypeScript** – Type safety

### Backend

- **Azure Functions** (Node.js) – Serverless API
- **TypeScript** – Type-safe code
- **@azure/arm-compute** – VM management
- **@azure/data-tables** – VM override state
- **@azure/identity** – Azure auth
- **Application Insights** – Monitoring & logging

## 🚀 Getting Started

### Prerequisites

- **Node.js 20** (Latest LTS recommended)
- **npm** or **yarn**
- **Azure Functions Core Tools** (for local API development)
- **Azure CLI** (optional, for deployment)
- Azure subscription with:
  - Virtual Machine
  - Storage Account
  - Application Insights instance

Output:

- Frontend: `.output/public/` (Static HTML/JS)
- Backend: `api/dist/` (Compiled Functions)
